# CLAUDE.md

프로젝트 관련 지시사항, 규칙, 선호도를 여기에 작성하세요.
Claude Code가 이 파일을 자동으로 읽고 참고합니다.

## 코드 품질

모든 작업 완료 후, Codex가 출력을 검토합니다.

## 업데이트 여부 확인

### notes/HOW-IT-WORKS.md
- 작업 완료 후, 프로젝트에 대해 새로 파악한 내용이 있으면 사용자에게 업데이트 여부를 확인한다
- 포함할 내용:
  - 기술 아키텍처 및 구조
  - 파일/폴더 구성과 연결 방식
  - 기술 결정 사항과 이유
  - 버그 해결 과정, 방법, 교훈
  - 주의사항 및 함정
  - 새로운 기술 학습
  - 좋은 엔지니어링 실천 방법
- 스타일: 재미있고 쉽게 읽을 수 있도록, 비유와 일화 활용

## 작업 성격별 모델/Effort 제안 프로세스

사용자가 작업을 요청하면, 작업 성격에 맞는 모델과 effort를 제안합니다.

### 작업 성격별 권장 설정

| 작업 성격 | 권장 모델 | 권장 Effort |
|----------|---------|----------|
| 새 기능 추가, 코드 리뷰/분석 | Sonnet | high |
| 버그 수정, 문서 작성, 파일 편집, 설정 변경 | Haiku | low |

### 프로세스

1. 사용자가 "뭐뭐 해줘"라고 요청
2. 작업 성격 판단 → 권장 모델/effort 제안
   - 예: "이 작업은 Sonnet/high를 권장합니다."
3. 변경을 원하면 `/model`, `/effort` 커맨드로 변경 후 다시 메시지를 보낸다
4. 변경하지 않으려면 **"이대로"** 를 입력
5. 변경 완료 신호 또는 "이대로" 확인 후 작업 개시
