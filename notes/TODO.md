## 🔄 CF) Superpowers 워크플로

| # | 단계 | 언제 활성화 | 핵심 동작 | 비고 |
|---|------|------------|----------|------|
| 1 | **브레인스토밍** | 코드 작성 전 | 질문으로 아이디어 정제 → 대안 탐색 → 섹션별 설계 검증 → 설계 문서 저장 | |
| 2 | **Git Worktree 생성** | 설계 승인 후 | 새 브랜치에 격리 작업공간 생성 → 프로젝트 셋업 → 테스트 기준선 확인 | 서브에이전트 1개·소규모 작업이면 오버스펙. `git checkout -b` 브랜치 생성만으로 대체 가능. Worktree는 여러 서브에이전트가 동시에 다른 폴더에서 작업할 때 진가 발휘 |
| 3 | **계획 작성** | 설계 승인 상태 | 작업을 2~5분 단위로 분해 → 파일 경로·전체 코드·검증 단계 명시 | |
| 4 | **서브에이전트 개발 / 계획 실행** | 계획 완료 후 | 태스크별 서브에이전트 디스패치 → 2단계 리뷰(스펙 준수 → 코드 품질) | |
| 5 | **테스트 주도 개발** | 구현 중 | RED(실패 테스트 작성) → GREEN(최소 코드) → REFACTOR → 커밋. 테스트 전 작성 코드 삭제 | |
| 6 | **코드 리뷰 요청** | 태스크 사이 | 계획 대비 구현 검토 → 심각도별 이슈 보고. Critical 이슈는 진행 차단 | |
| 7 | **개발 브랜치 완료** | 태스크 완료 후 | 테스트 통과 확인 → merge / PR / 유지 / 폐기 중 선택 → Worktree 정리 | |
